  fi
done

Detailed Chapter Playbook

CH1: Users, Groups, Authentication

  • Start with identity model: UID, GID, primary vs secondary groups, local users vs system users.
  • Create user accounts with home directories and secure shells, then enforce password policy.
  • Advanced: grant least-privilege sudo through user/group entries and validate with sudo -l.
useradd -m devops
passwd devops
usermod -aG wheel devops
chage -M 90 -W 7 devops
id devops
chage -l devops

CH2: File Management and Text Processing

  • Master create/copy/move/delete safely with recursive and force options only when needed.
  • Use wildcard, brace, variable, and command expansion to speed up repetitive work.
  • Advanced filtering combines grep, sort, tail/head for exam-grade quick extraction.
mkdir -p project/{docs,logs,backup}
cp -r project project-copy
grep -r "PermitRootLogin" /etc/ssh/
sort -u users.txt
tail -f /var/log/messages

CH3: Permissions, Ownership, ACL, Special Bits

  • Use symbolic and octal permissions correctly for files and directories.
  • Understand ownership flow with chown/chgrp in collaborative environments.
  • Advanced: SGID team directories, sticky-bit shared paths, ACL inheritance.
chmod 2770 /shared/project
chown root:devops /shared/project
setfacl -m u:qa:rwx /shared/project
setfacl -m d:g:devops:rwx /shared/project
getfacl /shared/project

CH4: Package and Repository Operations

  • Use dnf for install/remove/info/search/provides workflows.
  • Distinguish update vs upgrade behavior for production-safe decisions.
  • Advanced: custom repo setup and package history rollback awareness.
dnf repolist
dnf install -y httpd
dnf provides "*/semanage"
dnf history
rpm -q httpd

CH6: Process, Signals, Priority, Monitoring

  • Understand foreground/background job control and signal behavior.
  • Diagnose CPU/memory pressure using top, ps, uptime, and service logs.
  • Advanced: tune workload priority with nice/renice and tuned profiles.
ps aux | grep httpd
top
kill -15 PID
renice -n -5 -p PID
tuned-adm active

CH7/9: Partition, Filesystem, Mounting

  • Create partitions, format XFS/EXT4, mount temporarily and permanently.
  • Always prefer UUID in fstab to avoid device name drift.
  • Advanced: use mount -a and daemon-reload for safe validation before reboot.
lsblk -fp
mkfs.xfs /dev/vdb1
mkdir -p /data
blkid /dev/vdb1
# add UUID entry to /etc/fstab
mount -a

CH8: LVM and Swap (Lifecycle)

  • LVM chain: PV -> VG -> LV, then filesystem and mount.
  • Extend online with lvextend and grow filesystem appropriately.
  • Advanced: know safe reduction process for ext filesystems and deletion order.
pvcreate /dev/vdc1
vgcreate vgdata /dev/vdc1
lvcreate -n lvapp -L 4G vgdata
mkfs.xfs /dev/vgdata/lvapp
lvextend -L +1G /dev/vgdata/lvapp
xfs_growfs /dev/vgdata/lvapp

CH10: Shell Scripting and Automation

  • Write scripts with shebang and strict mode for reliability.
  • Automate repetitive checks (services, permissions, reports).
  • Advanced: combine find -exec with redirection for audit pipelines.
#!/usr/bin/env bash
set -euo pipefail
find /usr -perm /g+s -size +30k -size -45k > /root/usr-files
for s in sshd chronyd firewalld; do systemctl is-active --quiet "$s" || echo "$s down"; done

CH11: Task Scheduling

  • Use at for one-time tasks and crontab for recurring jobs.
  • Understand per-user vs system crontab format differences.
  • Advanced: apply cron allow/deny and anacron behavior for missed jobs.
at now + 10 minutes
crontab -e
crontab -l
cat /etc/crontab
ls /etc/cron.daily

CH11b: Logs and Time Synchronization

  • Track logs with journalctl and traditional /var/log files.
  • Configure timezone and chrony NTP client correctly.
  • Advanced: investigate authentication, cron, and service-specific failures quickly.
journalctl -xe
tail -f /var/log/secure
timedatectl
dnf install -y chrony
chronyc sources

CH12 + Networking: Interfaces, Routes, DNS

  • Set static IPv4 with nmcli and activate profile correctly.
  • Verify interface, route, DNS, and connectivity in sequence.
  • Advanced: troubleshoot from link layer to name resolution rapidly.
nmcli con show
nmcli con mod "Wired connection 1" ipv4.addresses 192.168.56.20/24
nmcli con mod "Wired connection 1" ipv4.gateway 192.168.56.1
nmcli con mod "Wired connection 1" ipv4.method manual
nmcli con up "Wired connection 1"
ip a; ip r; ping -c 3 8.8.8.8

CH15: SELinux and Firewall

  • Understand SELinux mode, context, type enforcement, and policy tools.
  • Use restorecon/semanage for permanent label and port policy fixes.
  • Advanced: integrate firewall zones/services with secure service deployment.
getenforce
sestatus
semanage fcontext -a -t httpd_sys_content_t "/srv/web(/.*)?"
restorecon -Rv /srv/web
semanage port -a -t http_port_t -p tcp 8080
firewall-cmd --add-service=http --permanent
firewall-cmd --reload

NFS: Manual, Persistent, and AutoFS

  • Mount exports manually for testing, then persist in fstab if always required.
  • Use autofs for on-demand mounts in larger environments.
  • Advanced: validate export visibility and mount options before exam submission.
dnf install -y nfs-utils autofs
showmount --exports server
mount -t nfs server:/export /mnt/nfs
umount /mnt/nfs
# configure /etc/fstab or autofs maps
systemctl enable --now autofs

Podman: Rootful/Rootless Containers

  • Search, pull, run, inspect, and remove images/containers.
  • Use port mapping, volume mounting with :Z, and exec for debugging.
  • Advanced: generate systemd units and enable linger for rootless auto-start.
podman search httpd
podman pull registry.access.redhat.com/ubi9/httpd-24
podman run -d --name web -p 8080:8080 -v /backup:/var/www/html:Z registry.access.redhat.com/ubi9/httpd-24
podman ps
podman generate systemd --name web --files
loginctl enable-linger username
