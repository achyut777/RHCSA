NFS and AutoFS

Mount remote storage manually, persistently, or on-demand with autofs based on workload requirements.

Back To Handbook Open Simulator
From Scratch (Foundation)
  • Install nfs-utils and verify export visibility
  • Mount and unmount NFS shares manually
  • Persist mounts in /etc/fstab when always needed
  • Understand direct vs indirect map approach in autofs
Advanced Execution
  • Use autofs for scalable on-demand enterprise mounts
  • Separate map definitions under /etc/auto.master.d
  • Troubleshoot stale mounts and timeout behavior
  • Validate server export options against client mount flags
Core Command Workflow
dnf install -y nfs-utils autofs
showmount --exports server
mount -t nfs server:/export /mnt/nfs
umount /mnt/nfs
systemctl enable --now autofs
automount -m
Validation and Troubleshooting
  • mount | grep nfs
  • systemctl status autofs
  • cat /etc/fstab
