CH6

Processes, Monitoring, and Priority

Diagnose system load and service behavior using process tools, signals, and priority tuning controls.

Back To Handbook Open Simulator
From Scratch (Foundation)
  • Use ps/top/pstree to inspect runtime state
  • Control jobs with fg/bg/jobs and signals
  • Terminate safely with SIGTERM before SIGKILL
  • Read load average relative to CPU count
Advanced Execution
  • Tune scheduling weight using nice and renice
  • Use pkill/killall for name-based control
  • Apply tuned profiles for workload optimization
  • Correlate process state with service logs quickly
Core Command Workflow
ps aux | grep sshd
top
kill -15 12345
pkill -9 rogueproc
renice -n -5 -p 12345
uptime
tuned-adm active
Validation and Troubleshooting
  • ps -l -p 12345
  • systemctl status tuned
  • journalctl -u sshd --since "-10 min"
