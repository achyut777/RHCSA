CH16

Podman Containers and systemd Integration

Deploy rootful and rootless containers with persistent service behavior and SELinux-safe data mounts.

Back To Handbook Open Simulator
From Scratch (Foundation)
  • Search/pull/run/list/stop/remove images and containers
  • Map host ports to container ports
  • Copy files and exec into running containers
  • Mount host volumes with :Z for SELinux relabeling
Advanced Execution
  • Generate systemd units for auto-start container services
  • Configure rootless startup with loginctl enable-linger
  • Build local images from Containerfile
  • Understand privileged port and session constraints for rootless mode
Core Command Workflow
podman search httpd
podman pull registry.access.redhat.com/ubi9/httpd-24
podman run -d --name web -p 8080:8080 -v /backup:/var/www/html:Z registry.access.redhat.com/ubi9/httpd-24
podman ps -a
podman generate systemd --name web --files
loginctl enable-linger username
Validation and Troubleshooting
  • systemctl status container-web.service
  • podman logs web
  • curl localhost:8080
